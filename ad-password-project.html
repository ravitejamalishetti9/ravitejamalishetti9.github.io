Active Directory Access Issue Resolution

The Problem

Network Before Directory Services

Before implementing Active Directory, the network relied on local user accounts on each machine, resulting in inconsistent access and management challenges.

  • HR staff blocked from accessing HR documents
  • Finance users viewing unauthorized folders
  • No centralized account management
üîí

My Solution

  1. Deployed Windows Server 2022 as Domain Controller (AD-DC1) for RAVI.local.
  2. Organized domain with OUs: HR, Finance, IT, and Workstations.
  3. Created users and security groups: HR_Read, Finance_edit, and IT_Admins.
  4. Configured shared folders with NTFS & share permissions (\\AD-DC1\HR-Shared).
  5. Applied GPOs for password policies and desktop restrictions.
